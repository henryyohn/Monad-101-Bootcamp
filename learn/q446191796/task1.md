## 第一章：走进 Web3 世界

1. 简单描述一下本地开发、部署合约的流程

本地开发和部署智能合约是一个系统化的过程，涉及多个步骤：
编写合约代码：使用文本编辑器或 IDE（如 Visual Studio Code）编写 Solidity 智能合约代码。

设置本地区块链：使用工具如 Ganache、Hardhat Network 或 Truffle 的开发网络创建本地区块链环境，用于测试。

编译合约：使用 Solidity 编译器（solc）将合约代码编译为可部署的字节码。

本地部署和测试：在本地区块链上部署合约，获取其地址，并使用测试脚本验证功能。

准备部署：确保合约在本地测试通过后，准备部署到测试网或主网。

部署到测试网：配置部署工具（如 Hardhat 或 Truffle）连接到测试网（如 monad 测试网），使用其 RPC URL 部署合约，并验证功能。

部署到主网：在测试网验证无误后，将合约部署到主网。

这个过程确保合约在部署前经过充分测试，减少上线后的风险。


2. 简单描述一下用户在使用一个 DApp 时与合约交互的流程                                                

打开 DApp：用户访问 DApp 的网站。

连接钱包：DApp 提示用户连接加密货币钱包（如 MetaMask），用户授权后连接。

执行操作：用户在 DApp 上执行操作，如转账、查看数据或触发合约功能。

DApp 与合约通信：DApp 使用合约地址和 ABI（应用二进制接口）向区块链上的智能合约发送请求。

合约处理请求：智能合约执行请求的功能，并返回结果或执行动作。

查看结果：DApp 将合约执行的结果展示给用户。

这个流程确保用户操作透明且安全，记录在区块链上。
