### 三、简答题

6. 请说明`require`、`assert`、`revert`三者的使用场景差异（从触发条件和Gas退还角度）
   require: 验证用户输入，退还未使用的gas
   assert:断言 验证代码逻辑，消耗所有gas
   revert:自定义错误处理，比较灵活 退还未使用的gas



7. 某合约同时继承A和B合约，两者都有`foo()`函数：

```solidity
contract C is A, B {
    function foo() override(A,B) {...}
}
```

实际执行时会调用哪个父合约的函数？为什么？
      执行C自己的函数 ， 
      因为C自己 覆写了foo函数 

8. 当使用`call`方法发送ETH时，以下两种写法有何本质区别？

```solidity
(1) addr.call{value: 1 ether}("")
(2) addr.transfer(1 ether)
```
      call是底层方法，灵活性高 默认转发所有剩余gas 
      transfer 是更高级的封装 固定2300gas
      call用来灵活处理一些场景 
      transfer简单安全的转账
